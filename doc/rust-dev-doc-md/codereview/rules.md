# Rust Code Review 规则

> 适用于所有架构模式（后端 / Tauri / WASM 前端）
> 最后更新：2026-04-11

---

## 一、Review Checklist

### P0 — 必须检查（阻塞合并）

#### 安全性
- [ ] 是否有 `unsafe` 块且缺少 SAFETY 注释
- [ ] 是否正确处理了用户输入（校验、转义）
- [ ] 是否有 SQL 注入风险（是否使用参数化查询）
- [ ] 是否有未初始化内存访问风险

#### 所有权与生命周期
- [ ] 是否有不必要的 `clone()`（应通过引用传递）
- [ ] 生命周期标注是否正确
- [ ] `Arc` 使用是否必要（是否能用引用）
- [ ] 是否有循环引用（`Rc<RefCell>` 或 `Arc<Mutex>` 循环）

#### 错误处理
- [ ] 是否有 `unwrap()` 在非测试代码中
- [ ] 是否有 `panic!()` 在可恢复错误场景
- [ ] 错误是否添加了上下文信息
- [ ] `Result` 是否正确传播

#### 并发安全
- [ ] async 代码是否使用了 `tokio::sync::Mutex`
- [ ] `Send`/`Sync` 约束是否正确
- [ ] 是否有数据竞争风险
- [ ] goroutine/task 是否有生命周期管理

### P1 — 重要检查

#### 性能
- [ ] 是否在循环中分配堆内存
- [ ] 是否有不必要的字符串分配
- [ ] 是否使用了正确的集合类型（Vec vs HashMap vs BTreeMap）
- [ ] 是否有潜在的 O(n²) 算法

#### 代码质量
- [ ] 命名是否符合 RFC 430
- [ ] public 符号是否有文档注释
- [ ] `match` 是否穷尽
- [ ] 函数长度是否合理（≤ 50 行）
- [ ] 是否有重复代码

### P2 — 建议检查

- [ ] 是否使用了惯用写法
- [ ] 是否可以简化为迭代器方法
- [ ] 测试是否覆盖
- [ ] 是否可以提取为独立 crate

## 二、Rust 特有 Review 关注点

| 关注点 | 说明 |
|--------|------|
| 所有权 | 优先借用，减少 clone |
| 生命周期 | 避免过度标注，使用 `'_` 省略 |
| 错误处理 | 库用 thiserror，应用用 anyhow |
| 并发 | async 用 tokio::sync，非 async 用 std::sync |
| trait 设计 | 保持小而精，使用泛型约束 |
| unsafe | 必须有 SAFETY 注释 |
