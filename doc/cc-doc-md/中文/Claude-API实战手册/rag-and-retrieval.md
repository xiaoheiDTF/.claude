# 能力篇：RAG 与检索增强

当“把资料直接塞进上下文”开始不够用时，这一组内容就会变得重要。它关注的不是某一种固定架构，而是帮你逐步回答一个更实际的问题：为了让回答更可靠，我该把检索做到什么程度。

## 先按问题复杂度选路径

### 只需要找到相关片段

先从基础 RAG 开始。它最适合回答“把正确内容找出来，再交给模型组织表达”的问题。

### 片段本身缺上下文

如果检索到的内容经常“看起来相关，但单独拿出来不够清楚”，就进入上下文化检索。重点不是把系统做复杂，而是让每个片段在被取回时更容易被理解。

### 你关心的是实体关系，而不是相似片段

当问题变成“谁和谁有关、关系如何推导、多个对象如何串起来”，就该考虑知识图谱，而不是继续堆向量检索。

### 召回和精度互相拉扯

如果你已经能检索到不少内容，但结果排序不稳定、噪声太多，就进入混合检索、重排和评测阶段。

## 这部分能帮你解决什么

- 给回答补上可追溯的上下文来源
- 让检索从“能用”走向“更稳定”
- 判断什么时候该继续强化 RAG，什么时候该换成更结构化的方案
- 为后续评测、引用和质量回归建立基础
## 推荐阅读顺序

- 先看 Retrieval Augmented Generation，建立最基础的检索心智模型
- 再看 Contextual Retrieval，解决片段脱离上下文的问题
- 如果你的问题是关系推理，再看 Knowledge Graph
- 进入更高要求时，再看混合检索、重排和评测相关内容
