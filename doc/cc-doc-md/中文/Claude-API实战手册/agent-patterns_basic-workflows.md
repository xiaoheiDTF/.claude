# Basic workflows（基础工作流）

如果你还没决定该用串行、并行，还是分阶段汇总，这一页最适合作为起点。它不是某个垂直场景的专门方案，而是一组可以复用到很多任务上的基础骨架。

可以把它理解成“工作流样板册”：先比较形状，再决定后面要不要加工具、检索、记忆或评测。

## 读的时候重点看

- 工作流骨架：拆分、串并行、汇总
- 插入 tools / 检索 / eval 的位置
- 用结构减少提示词长度
## 什么时候更适合用它

- 你还在判断任务更适合串行、并行还是分阶段推进
- 你想先找一个最小模式，再逐步加能力层
- 你需要的是通用骨架，而不是只适合某个领域的方案
## 如果你想本地复现

在本地环境已经准备好的前提下，可以对这个主题对应的示例做结构检查：

```bash
make test-notebooks NOTEBOOK=patterns/agents/basic_workflows.ipynb
```
