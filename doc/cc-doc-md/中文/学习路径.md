# 学习路径

如果你把 Claude 相关资料按仓库、按产品、按功能一个个去读，很容易越看越散。更高效的方式不是“先把所有资料都看完”，而是先明确你当前处在哪个阶段，再选择最合适的入口。

这页的目标，就是把站内现有内容整理成几条清晰的学习路径，让你知道：

- 先看什么
- 为什么先看它
- 看完以后下一步该去哪

## 路径一：先把 Claude Code 用起来

这条路径适合你还没有打算立刻做复杂集成，只是希望先让 Claude Code 真正进入日常开发。

### 推荐顺序

- Claude Code 快速开始
- Claude Code 入门指南
- 工作流
- 实用指南
### 这条路径的重点

- 先建立最小可用习惯，而不是一开始就研究所有配置
- 先学会把任务说清楚、把上下文给对、把验证做扎实
- 先让 Claude 帮你完成真实代码任务，再考虑更复杂的扩展
### 学完后的结果

你应该能够稳定完成以下事情：

- 让 Claude 阅读并修改现有代码
- 让 Claude 帮你排查构建和测试问题
- 用更少的提示词完成更高质量的改动
- 建立一套适合自己的基本工作流
## 路径二：做出第一个真实应用

这条路径适合你已经不满足于“在终端里问答”，而是想把 Claude 放进产品、页面或自动化流程里。

### 推荐顺序

- Quickstarts
- Cookbook
- Computer Use
- Extended Thinking
### 为什么这样排

`Quickstarts` 解决的是“先从哪种产品骨架开始”；`Cookbook` 解决的是“具体问题用哪种实现模式”；`Computer Use` 和 `Extended Thinking` 则帮助你进一步理解更强的交互和推理能力什么时候值得引入。

### 如果你的目标更具体

- 想做客服、知识助手：先从 Quickstarts 里的对话型路径开始，再补 Cookbook 中的检索与工具调用
- 想做数据分析和图表生成：先从分析型 Quickstarts 开始，再补 Cookbook 中的输出控制与评测
- 想做网页或桌面自动化：先读 Quickstarts 的自动化入口，再读 Computer Use
### 学完后的结果

你应该能够判断：

- 当前产品更适合从哪个官方起点开始
- 一个可运行示例离生产上线还差哪些工程层
- 哪些能力适合先验证，哪些能力不该过早引入
## 路径三：把经验沉淀成团队能力

这条路径适合你已经开始重复做同类任务，希望把“会话经验”变成“可复用的团队能力”。

### 推荐顺序

- Agent Skills
- Claude Code 进阶指南
- 插件系统
- 工作流
### 这条路径在解决什么问题

- 什么适合做成 Skill
- 什么适合做成插件
- 什么应该留在项目内配置里
- 如何把 Hooks、MCP、命令、Rules 组合成稳定工作流
### 一个实用判断标准

如果某件事只是你个人在一个项目里偶尔使用，先别急着封装。只有当它开始具备“重复出现、多人共用、需要版本化”的特征时，才值得进入 Skills、插件或团队配置层。

### 学完后的结果

你应该能更清楚地区分：

- 临时提示词 vs 可复用能力
- 项目内配置 vs 跨项目插件
- 单次经验 vs 团队工作流
## 路径四：从能力使用走向系统理解

这条路径适合已经会用 Claude Code，但想进一步理解它为什么这样设计，以及其中哪些模式值得迁移到自己的 Agent 系统里。

### 推荐顺序

- 源码分析总览
- Claude Code Agent Loop
- Claude Code 进阶指南
- 插件系统
### 这条路径的重点

- 理解工具、权限、上下文压缩、记忆、命令面、扩展面如何协同
- 区分“产品功能”与“Agent Harness 机制”
- 从成熟系统里提炼可以迁移的工程原则，而不是只看表面功能
### 更适合什么人

- 想自己做 Agent 平台或内部工具的人
- 想把 Claude Code 的经验迁移到团队工程体系的人
- 想理解为什么某些能力必须通过治理和边界来运行的人
## 一个简单的决策原则

如果你还在“先把事情做出来”的阶段，优先走 `Quickstarts + Cookbook + Claude Code`。

如果你已经在“把经验稳定下来”的阶段，优先走 `Agent Skills + Plugins + Workflows`。

如果你开始关心“系统为什么这样设计”，再进入 `Source Analysis + Agent Loop + Advanced`。

## 推荐的最小组合

### 个人开发者最小组合

- Claude Code 快速开始
- Quickstarts
- Cookbook
### 产品团队最小组合

- Quickstarts
- Cookbook
- Agent Skills
- 工作流
### 平台/架构团队最小组合

- 源码分析
- Claude Code 进阶指南
- 插件系统
- Agent Skills
