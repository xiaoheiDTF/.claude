# 文件管理

Claude Code 可以直接访问和操作工作区中的文件，无需手动上传下载。

## 文件引用

### @ 符号引用

最常用的方式，输入 `@` 后会弹出文件列表：

```
@报告.docx 帮我提取这份报告的核心观点
```

支持多文件引用：

```
@文件1.txt @文件2.txt 合并这两个文件的内容
```

### 路径引用
直接使用文件路径：

```
帮我分析 docs/会议记录/2024-01.md 这个文件
```

### 目录操作
对整个目录进行操作：

```
帮我统计 reports 目录下所有文件的字数
```

## 内容提取

### 提取摘要

```
@年度报告.pdf 生成一份200字的摘要
```

### 提取关键信息

```
@会议纪要.md 提取所有的决议事项和待办任务
```

### 提取特定数据

```
@合同.docx 提取所有的日期信息
```

结果示例：

- 合同签订日期：2024年1月15日
- 合同生效日期：2024年2月1日
- 项目截止日期：2024年12月31日
- 付款日期：分三期，2月1日、6月30日、12月31日
### 提取表格数据

```
@报告.pdf 提取文件中的所有表格
```

## 格式处理

### 统一文档格式

```
@文档1.docx @文档2.docx @文档3.docx 统一这些文档的格式：宋体、12号字、1.5倍行距
```

### 统一标题格式

```
@手册.md 统一标题格式：一级标题16号加粗居中，二级标题14号加粗左对齐
```

### 统一编号格式

```
@规范.docx 统一编号格式：一级用"一、二、三"，二级用"(1)(2)(3)"
```

## 格式转换

### 常见转换

```
@文档.docx 转换成 Markdown 格式
```

```
@数据.csv 转换成 Markdown 表格
```

```
@笔记.txt 转换成结构化的 Markdown 文档
```

### 批量转换

```
把 notes 目录下所有 .txt 文件转换成 .md 格式
```

## 文件创建

### 创建新文件

```
创建一个新文件 templates/会议通知模板.md，包含标准的会议通知格式
```

### 基于模板创建

```
参考 @模板.md 的格式，创建一份新的周报
```

## 批量操作

### 批量重命名

```
把 images 目录下的文件按"项目名_序号"格式重命名
```

### 批量内容替换

```
在 docs 目录下所有 .md 文件中，把"旧公司名"替换成"新公司名"
```

### 批量添加内容

```
在 reports 目录下所有文件开头添加版权声明
```

## 实战案例

### 案例：整理教学资料
场景：教师需要从多份教学文档中提取知识点，整理成教学大纲。

```
@教材1.pdf @教材2.pdf @教材3.pdf 提取这些教材的核心知识点，整理成一份12周的教学大纲
```

### 案例：合同信息汇总
场景：从多份合同中提取关键信息汇总。

```
@合同A.docx @合同B.docx @合同C.docx 提取每份合同的签约方、金额、有效期，汇总成表格
```

### 案例：文档标准化
场景：统一多份文档的格式规范。

```
@*.docx 检查并修复这些文档的格式问题：统一字体、修正编号、调整段落间距
```

## 注意事项

- 路径准确：确保文件路径正确，使用 / 作为分隔符
- 备份重要文件：大规模修改前建议先备份
- 分批处理：超大文件建议分批处理
- 验证结果：操作完成后检查结果是否符合预期
