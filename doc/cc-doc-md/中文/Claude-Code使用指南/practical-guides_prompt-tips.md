# 提示词技巧

掌握高效的提示词技巧，让 Claude Code 更准确地理解和执行你的需求。

## 清晰表达需求

### 结构化描述

好的提示词包含：任务目标、具体要求、输出格式。

**一般写法**：

```
帮我写个产品介绍
```

**更好的写法**：

```
帮我写一段智能手表的产品介绍：
- 核心卖点：健康监测、运动追踪、消息提醒
- 目标用户：年轻白领
- 风格：现代时尚
- 字数：200字左右
- 包含行动号召
```

### 提供上下文
说明背景信息，帮助 Claude 理解场景：

```
我是一名行政人员，需要给全公司发一封会议通知。
会议主题是Q1项目复盘，时间是下周三下午2点，地点是3楼会议室A。
请帮我写一封正式但不生硬的通知邮件。
```

### 指定约束条件
明确限制和要求：

```
把这篇文章缩写到500字以内，保留所有数据和结论，去掉案例描述
```

## 模板复用

### 创建提示词模板
把常用的提示词结构化保存：

**文案模板**：

```
请帮我写一段[类型]文案，关于[产品/服务]。
核心特点：[特点1]、[特点2]、[特点3]
目标受众：[受众]
风格要求：[风格]
字数要求：[字数]
```

**会议纪要模板**：

```
请把以下会议记录整理成正式的会议纪要：
[会议记录内容]

要求：
1. 提取会议要点
2. 整理决议事项
3. 列出待办任务（含负责人和截止日期）
4. 包含下次会议安排
```

**数据分析模板**：

```
请分析这份[数据类型]数据：
@[文件名]

分析要求：
1. 生成统计摘要
2. 分析数据趋势
3. 发现数据洞察
4. 提供改进建议
```

### 批量使用模板

```
使用产品介绍模板，分别为以下产品生成文案：
1. 无线耳机 - 主打降噪和续航
2. 便携音箱 - 主打音质和防水
3. 智能手环 - 主打健康监测
```

## 任务自动化

### 识别重复任务
常见可自动化的任务：

- 周报/月报撰写
- 会议纪要整理
- 数据统计报告
- 通知邮件撰写
- 文档格式统一
### 建立工作流

**周报自动化流程**：

- 收集数据
```
@本周任务.md @本周数据.xlsx 提取本周的工作完成情况和关键数据
```

- 生成周报
```
基于以上信息，生成周报，包含：
- 本周工作总结
- 数据分析
- 问题和建议
- 下周计划
```

- 格式调整
```
按公司周报模板格式调整
```

## 迭代优化

### 逐步细化
如果结果不理想，可以追加要求：

```
第一轮：帮我写一段产品介绍

第二轮：语气再活泼一些，加入一些比喻

第三轮：开头换一个更吸引人的切入点
```

### 提供反馈
告诉 Claude 哪里需要调整：

```
这个版本不错，但有几点需要修改：
1. 第二段太长，拆成两段
2. 价格信息放到最后
3. 加一句使用场景的描述
```

### 保存优秀结果
对满意的结果，记录使用的提示词，方便下次复用。

## 实战案例

### 案例：行政人员的日常通知

场景：需要发送各类通知（会议、活动、政策）。

**建立通知模板库**：

```
请帮我创建三个通知模板：会议通知、活动通知、政策通知。
每个模板包含必要的占位符，方便填充具体信息。
```

**使用模板**：

```
使用会议通知模板，填充以下信息：
- 会议主题：项目进度汇报会
- 时间：1月20日 14:00-16:00
- 地点：3楼会议室A
- 参会人员：项目组全体成员
- 议程：各模块进度汇报、问题讨论、下周计划
```

### 案例：内容创作者的批量生产
场景：需要为多个平台创作内容。

```
我有一篇产品评测文章，请帮我改编成：
1. 微信公众号版本（800字，图文并茂风格）
2. 小红书版本（300字，种草风格，带emoji）
3. 微博版本（140字以内，突出亮点）

原文：@评测原稿.md
```

## 常见问题
**Q: 结果太长/太短怎么办？**
A: 明确指定字数要求，如"200字左右"、“不超过500字”

**Q: 风格不对怎么办？**
A: 提供参考示例，或描述具体的风格特征

**Q: 格式不符合要求怎么办？**
A: 提供模板或明确说明格式规范

**Q: 内容不够专业怎么办？**
A: 提供专业术语、行业背景等上下文信息
