# 代码修改

让 Claude 按照你的意图精确修改代码需要一些技巧。本节介绍从规划到执行的完整工作流。

## 使用截图精确沟通

用 **Ctrl+V**（或 **Cmd+V**）直接粘贴截图：

```
> [粘贴 UI 截图]
> 这个按钮点击后没反应，修复它
```

**适用场景：**

- UI 布局问题
- 错误弹窗截图
- 设计稿对比
- 控制台错误截图
截图能传达文字难以描述的视觉信息，大幅提高沟通效率。

## 规划模式

对于复杂任务，使用规划模式让 Claude 先制定方案：

```
> /plan 重构用户认证模块，使用 JWT 替换 session
```

或在对话中要求规划：

```
> 帮我规划如何将这个单体应用拆分为微服务，先不要动代码
```

规划模式的优势：

- 在执行前审查方案
- 发现潜在问题
- 调整实现策略
- 确保理解一致
## 具体化你的指令

模糊的指令导致不确定的结果。

**避免：**

```
> 优化这个函数
```

**推荐：**

```
> 优化 processData 函数：
> 1. 减少数据库查询次数（当前每次循环查询一次）
> 2. 添加结果缓存
> 3. 保持向后兼容
```

**对比示例：**

| 模糊指令 | 具体指令 |
| --- | --- |
| 修复这个 bug | 修复 login.ts 第 42 行的空指针异常 |
| 添加验证 | 为邮箱字段添加格式验证和长度限制 |
| 改进性能 | 将 O(n²) 的嵌套循环优化为 O(n) |
| 重构代码 | 将 UserService 拆分为 UserAuthService 和 UserProfileService |

## 尽早纠正方向

发现 Claude 走错方向时，立即中断并纠正：

```
> [按 Escape]
> 不对，我需要的是 REST API 而不是 GraphQL
```

越早纠正，浪费的时间越少。不要等 Claude 完成再说"这不是我要的"。

## 让 Claude 先制定计划

对于非trivial的任务，先让 Claude 说明它打算怎么做：

```
> 在开始之前，先告诉我你打算如何实现这个功能
```

```
> 列出你需要修改的文件和原因
```

这样你可以在 Claude 动手前发现误解。

## 使用 Escape 中断

**Escape** 键是最重要的控制手段：

- 单击 Escape：中断当前操作
- 双击 Escape：返回历史编辑（修改之前的输入）
**中断后的选项：**

- 提供更多上下文
- 修改要求
- 让 Claude 换个方向
- 完全取消当前任务
## 双击 Escape 编辑历史

双击 **Escape** 可以回到之前的输入进行编辑：

```
> 实现用户登录功能
[Claude 开始执行]
[双击 Escape]
> 实现用户登录功能，使用 OAuth 2.0
```

这比重新输入整个指令更快。

## 让 Claude 撤销更改

如果修改不符合预期：

```
> 撤销刚才的更改
```

```
> 恢复 auth.ts 到修改前的状态
```

Claude 会使用 Git 或直接编辑来撤销更改。

## 分步执行复杂任务

将大任务拆分为小步骤，逐个确认：

```
> 第一步：创建数据库迁移文件
[确认后]
> 第二步：更新模型定义
[确认后]
> 第三步：修改 API 端点
```

这种方式让你在每个阶段都保持控制。

## 代码修改检查清单

执行代码修改时，确保：

- Claude 理解了任务背景
- 指令足够具体
- 复杂任务先规划
- 及时中断错误方向
- 修改后运行测试
- 审查生成的代码
