# 插件系统

插件适合把一套稳定能力打包成“可安装、可共享、可版本化”的扩展。无论你是想装现成能力，还是把团队流程沉淀下来，这一组文档都围绕三个实际问题展开：什么时候该用插件、从哪里安装、怎么确认它真的适合你的工作流。

## 什么时候该考虑插件

- 你希望一套命令、Agents、Skills 或 MCP 配置能跨项目复用
- 你希望团队成员装上以后就能按同一套流程工作
- 你希望能力可以版本化，而不是散落在聊天记录或临时脚本里
## 什么时候不一定需要插件

- 你只是在当前项目里试一个想法
- 你只需要几条本地命令或少量提示词
- 你还在频繁调整流程，暂时不值得封装
这类场景下，先用项目内的 `.claude/` 配置通常更轻、更快。

## 建议阅读顺序

如果你是第一次接触插件，建议按这个顺序阅读：

- 先看 先理解插件，建立基本心智模型
- 再看 发现和安装插件，把安装、验证和管理流程跑一遍
- 如果你还不确定先装哪一类，先看 按用途选择插件
- 如果要做技术选型，再看 选择插件来源
- 当现成插件不够用时，再进入 创建插件
## 插件和项目内配置的区别

| 方式 | 更适合什么场景 |
| --- | --- |
| 项目内配置 | 当前项目专用、快速实验、个人习惯 |
| 插件 | 跨项目复用、团队共享、长期维护 |

## 一个最短上手路径

```bash
/plugin marketplace list
/plugin install code-simplifier@claude-plugins-official
/plugin list
```

这条路径的重点不是“先装很多”，而是先装一个最贴近你工作流的小能力，观察它是否真的让日常任务更顺。
