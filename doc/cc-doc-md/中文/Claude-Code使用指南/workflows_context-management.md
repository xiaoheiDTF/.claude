# 上下文管理

Claude Code 的响应质量取决于它接收到的上下文。过多无关信息会稀释关键内容，导致响应偏离目标；精准的上下文则能让 Claude 快速理解并准确执行任务。

## 初始化项目理解

使用 `/init` 命令让 Claude 快速了解项目结构：

```
> /init
```

此命令会生成或更新 `CLAUDE.md` 文件，包含：

- 项目概述和技术栈
- 目录结构说明
- 常用命令和约定
- 重要文件位置
## CLAUDE.md 的作用

`CLAUDE.md` 是项目的"记忆文件"，Claude 会自动读取它来理解项目背景。

**创建 CLAUDE.md：**

```
> 为这个项目创建 CLAUDE.md
```

**包含的内容建议：**

- 项目构建和测试命令
- 代码风格和命名约定
- 架构决策和设计模式
- 常见问题和解决方案
## 保持专注

使用 `/clear` 命令清除当前对话上下文，开始新的任务：

```
> /clear
```

适用场景：

- 完成一个任务后开始新任务
- 对话中积累了太多无关信息
- 需要 Claude 重新评估代码库
## 明确指定文件

通过 Tab 补全直接引用文件，让 Claude 精确定位：

```
> 查看 src/auth/login.ts 中的验证逻辑
```

使用相对路径或绝对路径都可以，Tab 补全会帮助你快速输入。

## 提供外部资源

让 Claude 获取 URL 内容：

```
> 阅读 https://api.example.com/docs 并实现对应的客户端
```

Claude 会获取页面内容并基于此完成任务。

## 传递数据的四种方式

### 1. 复制粘贴

直接将内容粘贴到对话中：

```
> 这是错误日志，帮我分析：
Error: Connection refused at 127.0.0.1:5432
```

### 2. 管道输入
通过管道将命令输出传递给 Claude：

```bash
cat error.log | claude "分析这个错误日志"
```

```bash
git diff | claude "审查这些更改"
```

### 3. 让 Claude 使用工具
Claude 可以自己执行命令获取信息：

```
> 运行测试并分析失败原因
```

```
> 检查 npm 依赖是否有安全漏洞
```

### 4. 读取文件
明确指示 Claude 读取特定文件：

```
> 读取 config/database.yml 并解释配置项
```

## 中断和重定向
按 **Escape** 键可以中断 Claude 的当前操作。

**使用场景：**

- Claude 走错了方向
- 你想补充更多信息
- 需要调整任务要求
中断后，你可以：

- 提供更多上下文
- 修正指令
- 让 Claude 尝试不同方法
```
> [按 Escape 中断]
> 等等，我忘了说这是一个 TypeScript 项目，请使用类型注解
```

## 最佳实践

| 做法 | 原因 |
| --- | --- |
| 任务开始前运行 `/init` | 确保 Claude 了解项目 |
| 任务完成后使用 `/clear` | 避免上下文污染 |
| 明确指定相关文件 | 减少 Claude 的搜索时间 |
| 分阶段提供上下文 | 避免一次性信息过载 |
| 使用 Escape 及时纠正 | 防止 Claude 在错误方向上走太远 |
如果你想进一步把“同一个会话在不同阶段应该如何输出”固定下来，可以使用三种角色模板（开发/调研/审查）：[上下文角色：开发 / 调研 / 审查](https://claudecn.com/docs/claude-code/workflows/context-roles/)。
