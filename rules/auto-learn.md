# Proactive Skill Monitoring

当你在对话中检测到以下信号时，**主动**（不需要用户调用 /learn）进行干预。

---

## 信号一：用户不满意 Skill 输出

### 检测条件（满足任一即触发）

- 用户说 "不对"、"不是这样"、"重做"、"不好"、"改一下"、"不是我要的"、"又错了"
- 你使用某个 Skill 后，用户连续修正超过 2 次
- 用户手动改了 Skill 生成的结果，并说 "应该这样"
- 同一个任务用户让你重试了 2 次以上

### 响应动作

1. **不打断当前任务** — 先完成用户当前的要求
2. **在任务完成后的自然停顿点**，主动说：

```
我注意到 /<skill名> 的输出不太符合你的预期。
具体问题：<一句话描述问题>

要不要我记录下来？下次遇到类似情况我会自动调整。
```

3. 如果用户同意（或说"记下来"、"好"等），直接将修正写入 `.claude/skills/learn/corrections.md`，格式：

```markdown
## <skill名> 修正：<简短描述>

> 记录于 YYYY-MM-DD

**问题输出**: <Skill 生成了什么>
**期望输出**: <用户实际想要什么>
**根因分析**: <为什么会不匹配>
**修正方案**: <Skill 的 SKILL.md 应该加/改什么>
**已修改 Skill**: 否
```

4. 如果修正方案明确且改动小（加一条规则、改一段描述），**主动提议**："我可以直接改 `<skill名>/SKILL.md`，要改吗？"
5. 用户同意后用 Edit 工具修改 SKILL.md

---

## 信号二：重复操作模式

### 检测条件（满足任一即触发）

- 你在本次对话中执行了相同的工具组合 3 次以上（如 Grep→Read→Edit 反复出现）
- 用户连续 2+ 次要求做非常相似的事情
- 你发现自己在用"手动"方式完成某个可以自动化的流程

### 响应动作

1. **在下一个自然停顿点**，主动说：

```
我注意到我们反复在做：<描述重复的操作模式>
已经重复了 X 次。

要不要我：
1. 记下来，以后遇到类似情况自动按这个模式做？
2. 做成一个 Skill，以后一个命令搞定？
```

2. 如果用户选择"记下来"，写入对应的模式库文件：
   - 调试流程 → `.claude/skills/learn/debug.md`
   - 重构手法 → `.claude/skills/learn/refactor.md`
   - 架构决策 → `.claude/skills/learn/architecture.md`
   - 代码模板 → `.claude/skills/learn/snippets.md`

3. 如果用户选择"做成 Skill"，用 `/cc-architect` 的方法论创建新 Skill

---

## 信号三：新发现的工作偏好

### 检测条件

- 用户明确表达偏好："我喜欢..."、"我习惯..."、"以后都这样..."
- 用户纠正了你的风格/格式/命名习惯，且这种纠正出现了 2+ 次
- 用户说"别这样"、"不要这样"、"以后不要..."

### 响应动作

1. 直接说："好的，我记住了。把这条写入你的个人偏好库？"
2. 用户同意后，写入 `.claude/skills/learn/refactor.md`（编码风格偏好）或对应文件

---

## Hook 辅助数据

本项目配置了两个 Hook 自动采集数据：

- **session-track.sh**（PostToolUse）— 每次工具调用后记录工具名和目标文件到 `doc/session-tracking/`
- **session-end.sh**（Stop）— 会话结束时分析跟踪数据，检测：
  - 同一文件被编辑 3+ 次（返工信号）→ 写入 `pending-reviews.md`
  - 同一工具调用 10+ 次（重复信号）→ 写入 `pending-reviews.md`
  - Grep+Edit 循环 3+ 次（搜索-修复模式）→ 写入 `pending-reviews.md`

**当新会话开始时**，检查 `doc/session-tracking/pending-reviews.md` 是否有上次遗留的模式分析。如果有，在对话初期主动提及：

```
上次会话检测到以下模式：
<列出 pending-reviews.md 中的内容>
要不要回顾一下，把这些模式记下来？
```

## 关键约束

- **不频繁打扰** — 每个信号在单次对话中最多主动提醒 2 次
- **不在任务中途打断** — 只在自然停顿点（任务完成、切换话题时）提醒
- **不自动修改 Skill** — 修改 SKILL.md 必须用户明确同意
- **自动记录 corrections** — corrections.md 可以自动写入，这是安全的（只是笔记）
- **如果用户说"不用"、"算了"** — 停止本次对话的后续提醒
- **检查 pending-reviews** — 新会话开始时主动检查 Hook 产出的模式分析
