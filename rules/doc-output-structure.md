# 项目文档输出目录结构规范（Skill 外文档专用）

> **目标**: 统一 **Skill 规范之外** 的项目文档存放位置，防止 `doc/` 根目录文件堆积混乱
> **适用范围**: **非 Skill 流程** 产生的文档（如普通对话中 AI 直接生成的分析、总结、设计稿等）
> **重要前提**: 各 Skill 内部已有固定输出规范的文档，**继续沿用原 Skill 规范**，不受本规则约束

---

## 规则优先级说明

| 场景 | 遵循规范 | 示例路径 |
|------|---------|---------|
| Skill 内部生成的文档（`idea-to-req`、`impl-planner`、`code-implementer`、`web-research` 等） | **各 Skill 自身规范** | `03-plan/manifest.json`、`04-report/implementation.md` |
| 非 Skill 流程、普通对话中生成的项目文档 | **本规则**（`doc/当前任务/`） | `doc/路线优化调研/or-tools-vs-遗传算法.md` |

**判断依据**：如果当前操作是在某个 Skill 的明确工作流内（如用户调用了 `/idea-to-req`、`/impl-planner`，或处于 `code-implementer` 的某一步骤），则优先遵循该 Skill 的输出路径规范；否则适用本规则。

---

## 核心约束

### 约束一：Skill 外文档必须存放于 `doc/` 目录下

任何 **Skill 规范未覆盖** 的项目相关文档（设计文档、技术方案、调研报告、分析总结、会议纪要、API 文档等），**必须**输出到项目根目录的 `doc/` 文件夹下。

**例外情况**（不受此约束）：
- 根目录 `README.md`、`CLAUDE.md`、`AGENTS.md` 等顶层元数据文件
- 各业务模块内部的 `CLAUDE.md`（受 `claude-md-guardian.md` 规则约束，与源码放在一起）
- 构建产物或自动生成的日志（应进入 `target/`、`dist/`、`logs/` 等目录）
- **Skill 内部按自身规范输出的文档**

### 约束二：必须按当前任务分文件夹存放

在 `doc/` 目录下，**禁止**直接将 `.md` 文件堆积在根目录。

必须采用二级（或多级）目录结构：

```
doc/
├── <当前任务目录>/            ← 以当前任务/主题命名文件夹
│   ├── ai输出的md文件.md       ← AI 生成的文档
│   ├── xxx.md
│   └── ...
├── <另一任务目录>/
│   ├── aaa.md
│   └── bbb.md
└── ...
```

**目录命名规范**：
- 使用**中文或英文小写**，以**当前任务**命名，简洁清晰
- 优先复用已存在的同主题目录，避免重复创建含义相近的文件夹
- 若一个任务涉及多个子主题，可在任务目录下再分子目录

### 约束三：文件命名规范

- 使用有意义的文件名，能反映文档内容
- 统一使用 `.md` 扩展名
- 文件名中避免使用空格，可用 `-` 或 `_` 连接

## 正确与错误示例

### ✅ 正确示例（非 Skill 场景）

用户说："帮我调研一下 OR-Tools 和其他路线规划算法的对比"

```
doc/
├── 路线优化算法调研/
│   ├── or-tools-vs-遗传算法.md
│   └── 各算法性能对比.md
```

用户说："整理一下今天讨论的 API 设计"

```
doc/
├── api设计讨论/
│   └── 2026-04-23-api设计纪要.md
```

### ❌ 错误示例

```
doc/
├── or-tools-vs-遗传算法.md     ← 错误：直接放在 doc/ 根目录
├── api设计纪要.md              ← 错误：直接放在 doc/ 根目录
├── 临时笔记.md                  ← 错误：直接放在 doc/ 根目录，命名也不规范
└── ...
```

---

## 禁止行为

- **禁止**将 Skill 外文档输出到 `travel-agent-java/`、`travel-web/` 等业务源码目录中（模块内部的 `CLAUDE.md` 除外）
- **禁止**将 `.md` 文档直接堆积在 `doc/` 根目录下
- **禁止**创建含义模糊或过于宽泛的目录名（如 `doc/其他/`、`doc/临时/`、`doc/杂项/`）
- **禁止**以本规则覆盖 Skill 内部已定义的输出路径

---

## 执行检查点

在以下场景，**必须**检查文档输出位置是否符合本规则：

| 场景 | 检查动作 |
|------|---------|
| 普通对话中用户要求"写个文档"、"整理一下"、"调研一下" | 先判断是否处于 Skill 流程；若不是，按 `doc/<当前任务>/` 输出 |
| AI 主动生成的分析、总结、对比等非 Skill 产出物 | 确认输出路径为 `doc/<当前任务>/` |
| 发现 `doc/` 根目录下存在 `.md` 文件 | 主动提醒或迁移到合适的子目录中 |
| Skill 执行过程中需要额外出文档 | **优先遵循 Skill 自身规范**，不套用本规则 |
