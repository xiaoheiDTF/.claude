## 完整示例

### 输入（用户想法）

```
我想做一个旅游推荐功能，用户输入目的地后能自动生成路线
```

### 输出（README.md）

```markdown
# 旅游智能路线推荐

## 速览卡

**核心目标**: 用户输入目的地和偏好后，系统自动生成最优旅游路线
**规模**: 4 个核心功能 / 涉及 Java 后端、Python Agent 两个模块
**关键约束**: 依赖高德 API 获取 POI 数据；路线规划使用 OR-Tools 求解

---

## 1. 背景与动机

**问题现状**：用户需要手动搜索景点、规划路线，耗时且容易遗漏热门地点

**期望状态**：用户只需输入目的地和大致偏好，系统自动生成时间合理、路线优化的行程

---

## 2. 用户故事

### 故事 1：一键生成行程
- **角色**：游客
- **行为**：输入目的地、出行天数、预算后，点击生成
- **目的**：快速获得一条合理的旅游路线
- **验收**：系统返回包含每日景点、交通方式、预估费用的完整行程

### 故事 2：偏好定制
- **角色**：游客
- **行为**：选择偏好标签（美食/文化/自然/购物等）
- **目的**：让推荐结果更符合个人兴趣
- **验收**：生成路线中 70% 以上景点匹配用户偏好标签

---

## 3. 功能需求

### FR-01：目的地搜索与 POI 获取
- **描述**：用户输入目的地名称后，系统调用高德 API 获取该城市的热门景点、餐饮、住宿等 POI 数据
- **触发条件**：用户在搜索框输入目的地并确认
- **前置条件**：高德 API 可用
- **验收标准**：返回目的地城市不少于 20 个 POI，每个 POI 含名称、坐标、评分、类别
- **优先级**：P0

### FR-02：偏好采集
- **描述**：用户通过对话或标签选择，告知系统自己的旅游偏好（兴趣类型、预算范围、出行人数）
- **触发条件**：路线生成前
- **前置条件**：用户已登录
- **验收标准**：系统至少采集到目的地、天数、预算 3 项必填信息
- **优先级**：P0

### FR-03：路线求解
- **描述**：系统基于 POI 数据和用户偏好，使用 OR-Tools 求解最优多日路线
- **触发条件**：偏好采集完成，用户点击"生成路线"
- **前置条件**：POI 数据已获取，偏好已采集
- **验收标准**：返回的路线中，每日景点间驾车时间不超过 1 小时，总费用在预算 ±10% 内
- **优先级**：P0

### FR-04：路线展示与交互
- **描述**：用户查看生成的路线，可拖拽调整景点顺序、删除/替换景点，系统实时更新时间和费用
- **触发条件**：路线求解完成
- **前置条件**：路线已生成
- **验收标准**：用户调整后，系统在 2 秒内重新计算时间线并更新费用估算
- **优先级**：P1

---

## 4. 非功能需求

| 维度 | 要求 | 备注 |
|------|------|------|
| 性能 | 路线生成响应时间 < 5 秒 | 20 个 POI 以内 |
| 安全 | 用户偏好数据加密存储 | |
| 兼容性 | 支持移动端和 PC 端 | |

---

## 5. 约束与假设

### 技术约束
- 必须使用高德 API 获取 POI（公司已采购）
- 路线求解必须使用 OR-Tools（已有求解器框架）
- 前端使用 Vue/React（项目技术栈）

### 假设
- [假设] 用户首次使用时不需要注册，可先体验再登录
- [假设] 高德 API 每日调用额度足够支撑初期用户量
- [假设] 路线调整时不需要重新调用 OR-Tools，前端本地计算即可

---

## 6. 范围定义

### 包含
- ✅ 目的地 POI 搜索
- ✅ 偏好采集对话
- ✅ OR-Tools 路线求解
- ✅ 路线展示与调整

### 不包含
- ❌ 酒店/机票预订 — 原因：一期只做路线规划
- ❌ 实时天气集成 — 原因：后续版本加入
- ❌ 社交分享 — 原因：非核心路径

---

## 7. 影响范围

| 子项目/模块 | 影响类型 | 说明 |
|------------|---------|------|
| travel-agent (Python) | 新增 | 偏好采集对话流、意图解析 |
| travel-server (Java) | 新增 | OR-Tools 路线求解、POI 数据获取 |
| travel-web (前端) | 新增 | 路线展示组件、偏好选择 UI |

---

## 8. 建议与扩展

- 后续可加入"多方案对比"：一次生成 2-3 条不同风格的路线供用户选择
- 可接入天气 API，雨天自动优先推荐室内景点
- 可加入"行程分享"功能，支持生成图片分享到社交平台
```
