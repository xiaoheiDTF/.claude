# impl-planner 参考模板

本文件包含 impl-planner 技能使用的所有输出文件模板，供生成计划时按需查阅。

---

## 模块 CLAUDE.md 模板

为计划中每个新建目录生成一个 CLAUDE.md，内容结构如下：

```markdown
# <模块名称> 模块

> 位置: `<相对项目根目录的路径>/`
> 关联计划: <计划保存目录名>
> 生成时间: YYYY-MM-DD HH:mm

## 简介

<一句话描述该模块的职责，从拆解报告的模块描述中提取>

## 目录结构

\```
<目录名>/
├── CLAUDE.md              ← 本文件
├── <file1>.<ext>          ← Step X: <一句话职责>
├── <file2>.<ext>          ← Step X: <一句话职责>
└── index.<ext>            ← Step X: 统一导出入口（如需要）
\```

## 文件创建要求

- 文件命名: <该语言的命名规范，如 kebab-case / PascalCase>
- 每个文件一个职责
- 导出方式: <具名导出 / 默认导出 / 等>
- <其他从项目 CLAUDE.md 和语言规范中提取的约定>

## 代码规范

- 遵循 `.claude/rules/<语言>.md` 中的编码规范
- 关键约束:
  - <从对应语言 Rules 中提取最相关的 3-5 条约束>
  - <从项目 CLAUDE.md 中提取的模块特定约束>
```

**内容填充规则**：
- **位置**：相对项目根目录的路径（如 `frontend/src/core/browser/adapter/`）
- **简介**：从拆解报告中该模块的描述提取，一句话
- **目录结构**：从计划中所有涉及该目录的 Step 汇总，每个文件标注来自哪个 Step
- **文件创建要求**：从项目级 CLAUDE.md 和对应语言 Rules 中提取命名和文件组织约定
- **代码规范**：引用对应的语言 Rules 文件，并提取与该模块最相关的 3-5 条关键约束（不要把整个 Rules 复制进来）

## manifest.json 模板

```json
{
  "type": "impl-planner",
  "version": "1.1",
  "generated_at": "YYYY-MM-DD HH:mm",
  "source_breakdown": "<关联的拆解报告目录名>",
  "source_reuse": "<关联的复用报告目录名>",
  "steps": [
    { "id": "S01", "name": "<步骤名>", "file": "S01-<简称>.md", "strategy": "自己写|编排组合|直接用", "target": "<目标文件路径>", "depends": [] }
  ],
  "steps_count": X,
  "languages": ["typescript", "python"],
  "acceptance_file": "ACCEPTANCE.md",
  "acceptance_scenarios": { "p0": X, "p1": X, "p2": X, "total": X },
  "new_dirs": [
    { "path": "<新建目录相对路径>", "has_claude_md": true }
  ],
  "modify_boundary": {
    "writable": ["<可修改目录>"],
    "readonly": ["<只读目录>"]
  }
}
```

**用途**：`code-implementer` 读取此文件，快速获取步骤顺序、策略、目标文件、依赖关系和修改边界，无需解析 README.md。
