# 代码复用报告模板参考

本文件包含 code-reuse-finder Skill 输出文件所使用的模板。Skill 执行时按需引用对应模板生成报告。

---

## README.md 模板

```markdown
# 代码复用参考建议

> 生成时间: YYYY-MM-DD HH:mm
> 关联拆解报告: <目录名>
> 原始需求: <需求简述>

---

## 目录

| 文件 | 内容 |
|------|------|
| 本文件 | 速览卡 + 复用密度统计 + 模块概览 |
| [R0-<简称>.md](./R0-<简称>.md) | <模块〇名称> — 复用详情 |
| [R1-<简称>.md](./R1-<简称>.md) | <模块一名称> — 复用详情 |
| ... | ... |
| [SUMMARY.md](./SUMMARY.md) | 复用总览表 + 建议 |

---

## 速览卡

**统计**: 直接能用 X 个 | 编排组合 X 个 | 自己写 X 个 | 复用率 X%

1. <最大复用资产是什么>

## 模块复用详情文件模板

每个模块一个文件（R0-xxx.md ~ Rn-xxx.md），包含该模块所有功能单元的复用查找结果。

```markdown
# <模块编号>、<模块名称> — 复用分析

> 模块目标: <一句话>
> 复用率: X/X 功能可复用（X%）

---

## 功能单元复用总览

| # | 功能单元 | 建议 | 复用代码位置 |
|---|---------|------|------------|
| <编号>.1 | <名称> | ✅ 直接用 | `file.ts:10-25` |
| <编号>.2 | <名称> | 🔧 编排组合 | `file.ts:30-40` + `file2.ts:5-15` |
| <编号>.3 | <名称> | 🆕 自己写 | — |

---

## 复用详情

<details>
<summary><编号>.1 <功能单元名称> — ✅ 直接用</summary>

**复用代码**: `file.ts:10-25`

**说明**: <怎么用这个现有代码>

</details>

---

<details>
<summary><编号>.2 <功能单元名称> — 🔧 编排组合</summary>

**复用代码**:
- `file.ts:30-40` — <函数名>，功能是 <简述>
- `file2.ts:5-15` — <函数名>，功能是 <简述>

**编排方式**: <先调用 A 获取 X，再调用 B 传入 X 得到 Y>

</details>

---

<details>
<summary><编号>.3 <功能单元名称> — 🆕 自己写</summary>

**原因**: <为什么不能复用>

**参考**: <可参考的现有代码结构或设计文档（如有）>

</details>
```

**注意**：
- 每个功能单元用 `<details>` 折叠块，默认收起，需要时点击展开
- **不内嵌完整代码片段**。复用代码只标注文件路径+行号范围 + 一句话说明。读者需要看代码时自己打开源文件
- "自己写"的功能单元也要折叠展开说明原因
- 模块复用总览表格提供快速扫读视图
- 使用 emoji 标记建议类型（✅ 直接用 / 🔧 编排组合 / 🆕 自己写）

| 0.2 | <名称> | 〇 | 编排组合 | <调用顺序> | `file.ts:30-40` |
| 0.3 | <名称> | 〇 | 自己写 | 无可复用 | — |
| ... | ... | ... | ... | ... | ... |

---

## 风险提示

- <复用现有代码可能遇到的风险>

---

## 遗漏补全

- <有现有代码可复用但拆解报告中未提及的功能>

---

## 简化建议

- <可以用更简单的编排方式实现的部分>
```

## manifest.json 模板

```json
{
  "type": "code-reuse-finder",
  "version": "1.0",
  "generated_at": "YYYY-MM-DD HH:mm",
  "source_breakdown": "<关联的拆解报告目录名>",
  "modules": [
    { "id": "M0", "short": "<模块简称>", "file": "R0-<简称>.md", "direct": X, "compose": X, "new": X, "reuse_rate": "X%" }
  ],
  "total": { "direct": X, "compose": X, "new": X, "reuse_rate": "X%" },
  "languages": ["typescript", "python"],
  "reuse_files": [
    { "path": "<复用代码相对路径>", "lines": "10-25", "function": "<函数名>" }
  ]
}
```

**用途**：下游 Skill（impl-planner）读取此文件，快速获取复用统计、涉及语言和所有复用代码位置，无需解析 Markdown。
