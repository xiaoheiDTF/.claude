## 大规模批量规范化 — 分步执行 + Todo 追踪

> 沉淀于 2026-04-13，来源：全项目 CLAUDE.md 缺失/不完整扫描与修复，涉及 30 个目录

**通用场景**: 项目已有大量目录缺少规范文档/配置文件时，需要批量生成和补全
**识别信号**: 扫描发现 10+ 个缺失或不完整的文件，直接逐个处理容易遗漏或重复
**通用做法**:
  1. **先全量扫描** — 用检查模式获取完整的缺失/不完整/过时列表
  2. **分类建 Todo** — 按操作类型分组（生成缺失、修复过时、补全不完整、初始化索引），每组一个 Todo 项
  3. **按顺序执行** — 生成缺失 → 修复过时 → 补全不完整 → 初始化 registry → 生成索引
  4. **内容定制** — 每类文件根据上下文定制内容（引用语言规则、匹配目录职责），不套用统一模板
  5. **最终报告** — 输出统计和详细列表，确认无遗漏
**原因**: 大规模操作如果逐个处理容易丢失进度和遗漏文件。分类分步执行让每步可验证、可回退
**避坑**: 不要为所有文件套用同一个模板 — 不同目录的职责不同，代码规范和 Review 检查清单应该针对性编写
**适用举例**: 批量添加 ESLint 配置到多个子项目、批量补充 API 文档、批量更新依赖版本并同步 lockfile、批量添加 .gitignore 规则

**项目场景**: zwyAi 项目 30 个含代码目录中，4 个缺失 CLAUDE.md、1 个目录结构过时、25 个缺少"代码规范"和"Review 检查清单"段落、module-registry 完全不存在
